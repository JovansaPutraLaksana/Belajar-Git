CATATAN GIT DASAR (REPOSITORY LOCAL)
===================================

Catatan ini membahas penggunaan Git secara LOCAL
(tanpa remote repository seperti GitHub).


--------------------------------------------------
1. KONSEP DASAR REPOSITORY LOCAL
--------------------------------------------------

Repository Local:
- Repository Git yang hanya ada di komputer pribadi
- Belum terhubung dengan GitHub / GitLab / Remote repository
- Semua perubahan hanya tersimpan secara lokal


--------------------------------------------------
2. THE THREE STATES (3 STATUS FILE DI GIT)
--------------------------------------------------

1. Modified
   - File telah berubah
   - Perubahan BELUM disiapkan untuk disimpan ke repository
   - Status: masih di Working Directory

2. Staged
   - File telah ditandai untuk disimpan
   - Perubahan sudah masuk ke Staging Area
   - Menunggu commit

3. Committed
   - File sudah tersimpan permanen di repository
   - Tercatat sebagai snapshot (riwayat)


--------------------------------------------------
3. THREE SECTION (3 AREA DI GIT)
--------------------------------------------------

1. Working Directory
   - Tempat kita bekerja
   - File bisa dibuat, diedit, dihapus
   - Perubahan belum dicatat Git

2. Staging Area
   - Area persiapan sebelum commit
   - Git tahu file mana saja yang akan disimpan

3. Repository
   - Tempat penyimpanan snapshot (riwayat perubahan)
   - Setiap commit = satu snapshot


--------------------------------------------------
4. WORKFLOW DASAR GIT
--------------------------------------------------

Alur kerja Git:

Working Directory
        |
        | (git add)
        v
Staging Area
        |
        | (git commit)
        v
Repository


--------------------------------------------------
5. PERINTAH DASAR CMD / TERMINAL
--------------------------------------------------

cd <nama folder>
- Masuk ke folder tertentu

cd ..
- Mundur satu level folder

mkdir <nama folder>
- Membuat folder baru

dir
- Melihat isi folder (file normal)

dir /a
- Melihat semua file termasuk hidden

dir /a:h
- Melihat file hidden saja


--------------------------------------------------
6. MEMBUAT REPOSITORY GIT
--------------------------------------------------

Perintah:
git init

Penjelasan:
- Membuat repository Git baru
- Akan muncul folder tersembunyi .git
- Folder ini berisi semua data Git


--------------------------------------------------
7. MELIHAT STATUS PERUBAHAN
--------------------------------------------------

Perintah:
git status

Keterangan warna:
- Merah  : perubahan di Working Directory
- Hijau  : perubahan di Staging Area


--------------------------------------------------
8. MENAMBAHKAN FILE KE STAGING AREA
--------------------------------------------------

Menambahkan semua file:
git add .

Menambahkan file tertentu:
git add <nama file>

Catatan:
- Case sensitive
- File harus ada di Working Directory


--------------------------------------------------
9. MENYIMPAN PERUBAHAN (COMMIT)
--------------------------------------------------

Perintah:
git commit -m "<pesan keterangan>"

Penjelasan:
- Menyimpan snapshot ke Repository
- Pesan commit harus jelas dan singkat


--------------------------------------------------
10. MELIHAT PERBEDAAN FILE (DIFF)
--------------------------------------------------

Melihat perubahan Working Directory:
git diff

Membandingkan commit lama dengan HEAD:
git diff <hash> HEAD

Menggunakan VS Code (Diff Tool):
git difftool <hash1> <hash2>

Catatan:
- Difftool butuh konfigurasi VS Code terlebih dahulu


--------------------------------------------------
11. MENGHAPUS & MEMBATALKAN PERUBAHAN
--------------------------------------------------

Menghapus file untracked (PERMANEN):
git clean -f

Membatalkan perubahan file di Working Directory:
git restore <nama file>

Mengembalikan file dari Staging ke Working Directory:
git restore --staged <nama file>


--------------------------------------------------
12. MELIHAT RIWAYAT COMMIT
--------------------------------------------------

Melihat semua commit:
git log

Tampilan singkat:
git log --oneline

Dengan graph branch:
git log --oneline --graph

Melihat detail commit:
git show <hash>


--------------------------------------------------
13. RESET COMMIT (MEMINDAHKAN HEAD)
--------------------------------------------------

Perintah:
git reset <mode> <hash>

Mode reset:
--soft
- HEAD pindah
- Staging Area & Working Directory aman

--mixed (default)
- HEAD pindah
- Staging Area di-reset
- Working Directory aman

--hard
- HEAD pindah
- Staging Area & Working Directory dihapus

Catatan penting:
- Jika belum commit baru, reset masih bisa dibatalkan
  jika masih ingat hash HEAD sebelumnya


--------------------------------------------------
14. AMEND COMMIT
--------------------------------------------------

Perintah:
git commit --amend -m "<pesan>"

Penjelasan:
- Mengubah commit terakhir
- Bisa untuk:
  - Ganti pesan commit
  - Menambahkan file yang lupa di-commit


--------------------------------------------------
15. MENGAMBIL FILE DARI COMMIT LAMA
--------------------------------------------------

Mengambil satu file:
git checkout <hash> -- <nama file>

Mengambil semua file:
git checkout <hash>

Kembali ke branch aktif:
git checkout <nama branch>


--------------------------------------------------
16. MELIHAT BRANCH AKTIF
--------------------------------------------------

Perintah:
git branch --show-current


--------------------------------------------------
17. REVERT COMMIT
--------------------------------------------------

Perintah:
git revert <hash>

Penjelasan:
- Membatalkan commit dengan commit BARU
- Aman untuk histori
- Tidak memindahkan HEAD ke belakang


--------------------------------------------------
18. GITIGNORE
--------------------------------------------------

File:
.gitignore

Contoh isi:
Ignore folder:
<nama folder>/

Ignore extension:
*.log
*.tmp

Ignore file tertentu:
.env
config.json


--------------------------------------------------
19. BLAME
--------------------------------------------------

Perintah:
git blame <nama file>

Penjelasan:
- Melihat siapa yang menambahkan baris kode
- Berguna untuk tracking perubahan
